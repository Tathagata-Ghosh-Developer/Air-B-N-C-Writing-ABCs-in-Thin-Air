# Air-Writing Digit Recognition (IMU + Vision Ensemble)

A lightweight, edge-deployable air-writing digit recognition system that fuses IMU motion data and vision-based trajectory images. The IMU branch runs a Random Forest on handcrafted features, while the vision branch uses a GhostNet-inspired CNN with ECA, optimized for Raspberry Pi inference.

## Highlights
- Multimodal fusion: IMU + vision for robust predictions
- Edge-friendly models for Arduino Nano 33 BLE Sense and Raspberry Pi 4
- End-to-end pipeline: data collection → training → deployment

## Dataset

## Reproducibility — step-by-step
### 1) IMU data collection (Arduino + host)
1. Flash src/data_collection/main.cpp to Arduino Nano 33 BLE Sense.
2. On the host, edit PORT in src/data_collection/collect_imu.py.
3. Run the collector:
   - python src/data_collection/collect_imu.py
4. Follow the prompt to label digits and collect CSV sessions.

### 2) Vision trajectory collection (Raspberry Pi)
1. Run the Streamlit UI:
   - streamlit run src/data_collection/streamlit_dashboard.py
2. Record each digit trajectory; PNGs are saved per class.

### 3) Train IMU model
Open src/training_pipeline/IMU_Train.ipynb and run the cells in order.
This exports rf_model.h and scaler_params.h for embedded inference.

### 4) Train vision model
Open src/training_pipeline/trajectory_cv_pipeline.ipynb and run the cells.
This exports FP32 and INT8 artifacts and ONNX models for Raspberry Pi.

### 5) Raspberry Pi inference
Use src/inference/rpi_pytorch_inference.py to run inference on trajectory images.
